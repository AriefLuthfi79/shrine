Plugins

Attachment

activerecord:
Provides ActiveRecord integration, adding callbacks and validations.
column:
Provides integration for handling attachments on a database column.
entity:
Provides integration for handling attachments on an immutable struct.
model:
Provides integration for handling attachments on a mutable struct.
sequel:
Provides Sequel integration, adding callbacks and validations.

Flow

backgrounding:
Adds the ability to put storing and deleting into a background job.
presign_endpoint:
Provides a Rack application that generates signatures for direct uploads to specified storage.
upload_endpoint:
Provides a Rack application that forwards file uploads to specified storage.

Processing

derivation_endpoint:
Provides a Rack application for processing uploaded files on-the-fly.
derivatives:
Allows storing a collection of processed files alongside the main file.

Source

data_uri:
Adds ability to upload files via a data URI.
rack_file:
Adds the ability to transform a Rack uploaded file hash into an IO-like object.
remote_url:
Allows your models to accept attachments as remote URLs.

Validation

remove_invalid:
Automatically deletes and deassigns a new assigned file if it's invalid.
validation:
Provides generic file validation interface.
validation_helpers:
Gives you convenient methods for validating file's metadata.

Metadata

add_metadata:
Allows extracting additional metadata values.
determine_mime_type:
Determine the actual MIME type from file contents.
infer_extension:
Deduce appropriate file extension based on the MIME type.
metadata_attributes:
Allows saving selected metadata values into separate record columns.
refresh_metadata:
Allows re-extracting metadata of an uploaded file.
restore_cached_data:
Re-extracts cached file's metadata on assignment.
signature:
Allows you to calculate a SHA{1,256,384,512}/MD5/CRC32 hash of a file.
store_dimensions:
Allows you to store image dimensions as additional metadata.

Downloading

download_endpoint:
Provides a Rack application through which uploaded files can be downloaded.
rack_response:
Adds the ability to convert a Shrine::UploadedFile object into a Rack response.
tempfile:
Makes it easier to reuse a single copy of an uploaded file on disk.

Form

cached_attachment_data:
Adds a model method for the hidden form cache field.
form_assign:
Allows attaching file from form params without a form object.
remove_attachment:
Allows removing the attachment via a form field.

Settings

default_storage:
Allows you to change default temporary and permanent storage.
default_url:
Allows setting the URL which will be returned on missing attachments.
dynamic_storage:
Allows you to define storages dynamically based on their identifier.
multi_cache:
Allows the attacher to accept files from additional temporary storages.
pretty_location:
Generates a more organized directory structure on the storage.
upload_options:
Allows setting default storage-specific upload options.
url_options:
Allows setting default storage-specific URL options.

Other

atomic_helpers:
Provides methods for retrieving and persisting attachments in a concurrency-safe way.
included:
Allows the included attachments to define additional methods on the model.
instrumentation:
Publishes and logs events for some main operations.
keep_files:
Prevents file deletion when replacing or removing attachment.
mirroring:
Enables replicating uploads and deletes to other storages.
